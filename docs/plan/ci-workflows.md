# Plan: CI Workflows

Three automation workflows ported from dakota (Bluefin):

- **`track-bst-sources`** ‚Äî daily `bst source track`, opens PRs for source updates
- **`cache-warm`** ‚Äî keeps the GitHub/Blacksmith Actions BST cache hot

`track-next-junctions` is not needed (krytis has no rolling `next` branch).

## Approach: native BST in CI

CI installs BuildStream natively via `uv` (the `uv.lock` added by this plan)
rather than pulling the upstream bst2 podman container. This follows
zirconium-hawaii's CI pattern:

- `bst source track` needs no sandbox ‚Äî just network access. No container pull.
- `bst build` (cache-warm) needs bubblewrap; apt-installed on the runner.
- No CAS push in the foreseeable future ‚Äî no credentials needed. Cache relies on
  GitHub/Blacksmith `actions/cache` only.

See `docs/plan/native-bst-local-dev.md` for the parallel plan to bring native
BST into local dev as well.

## `track-bst-sources.yml`

**Trigger:** `schedule: '0 6 * * *'` + `workflow_dispatch` (group input: `all |
manual-merge | linux-cachyos | core-junctions`)

**Runners:** `ubuntu-24.04` (lightweight ‚Äî no build, just `bst source track`)

**Three jobs:**

### `track` (matrix)

Tracks git-sourced elements via `uv run bst source track`. Single entry:

| group | element | branch | notes |
|-------|---------|--------|-------|
| `manual-merge` | `core/bootc.bst` | `auto/track-bootc` | Cargo2 + git; human review required |

Steps follow dakota's pattern: gate ‚Üí Mergeraptor token ‚Üí checkout ‚Üí `jdx/mise-action` ‚Üí
`uv run bst source track` ‚Üí SHA-comparison diff check (skip format-only
normalizations) ‚Üí create/update PR. No auto-merge.

### `track-linux-cachyos`

Runs `mise run kernel-update` (pure bash; fetches pacman DB, updates
`elements/core/linux-cachyos.bst` version/pkgrel/sha256 in place). Does not
invoke BST at all. Creates a PR to `auto/track-linux-cachyos` if changed. No
auto-merge ‚Äî kernel bumps require validation.

### `track-core-junctions`

Tracks `gnome-build-meta.bst` and `freedesktop-sdk.bst` atomically in a single
commit. These must land together because `freedesktop-sdk.bst` overrides
reference `gnome-build-meta.bst` elements directly; a partial update breaks the
element graph. No auto-merge ‚Äî junction bumps can break downstream elements.

Branch: `auto/track-core-junctions`.

## `cache-warm.yml`

**Trigger:** `schedule: '0 6 * * 1-5'` (weekday mornings) + `workflow_dispatch`

**Runner:** Blacksmith (heavy build ‚Äî needs RAM and disk for the full stack)

**Concurrency:** `krytis-cache-warm`, `cancel-in-progress: false`

**Purpose:** build `oci/krytis/image.bst` daily so the GitHub/Blacksmith
`actions/cache` (`~/.cache/buildstream`) stays warm. PR builds restore this
cache and get hits instead of cold-start rebuilds.

**Key steps:**
1. `jdx/mise-action@v2` ‚Äî installs Python + uv, auto-runs `uv sync` via
   `[deps.uv]`
2. `mise run generate-image-version`
3. Write `~/.config/buildstream.conf` (scheduler + logging settings; no auth
   section ‚Äî read-only CAS access is configured in `project.conf`)
4. `actions/cache` restore: key on `gnome-build-meta.bst` + `freedesktop-sdk.bst`
   + `uv.lock`
5. `apt-get install bubblewrap lzip xz-utils bzip2 gzip`
6. `sysctl -w kernel.apparmor_restrict_unprivileged_userns=0`
7. `uv run bst build oci/krytis/image.bst` ‚Äî non-blocking (`set +e`, exit 0 on
   failure with `::warning::`)

## Differences from dakota

Dakota's workflows are the reference implementation. Key adaptations:

- **No podman/bst2 container in CI** ‚Äî native BST via uv (see above)
- **No BST2 image pin check** ‚Äî replaced by `uv.lock` reproducibility
- **No `projectbluefin/actions/setup-runner`** ‚Äî inline equivalent steps
- **No `generate-bst-ci-config` action** ‚Äî no CAS push means no credentials to
  inject
- **No `track-tarballs` job** ‚Äî no tarball-sourced elements in krytis
- **No auto-merge group** ‚Äî krytis has no elements suitable for auto-merge today
- **No `next` branch job** ‚Äî see track-next-junctions note above
- **Blacksmith runners** for cache-warm; `ubuntu-24.04` for tracking jobs

## Python dependency notes

Mirrors zirconium-hawaii's package set (dropping `pgpy13`, which is specific to
Endless OS secure boot signing):

- `click==8.2.1` ‚Äî pinned; BST 2.5.x breaks with Click 8.3.0. Watch for a new
  BST release that resolves this.
- `dulwich==0.24.0` ‚Äî pinned; this library breaks API in patch releases.
- Both pins should be revisited when updating `buildstream` in `pyproject.toml`.
